# 一篇学习关于C++的日记

这里是学习C++学昏了头的W先生。

这篇日记不涉及任何C++具体内容，只是个人的一些感想。

# 事情的起因

Verilog的设计方式一直是被吐槽的内容，而VHDL又过于麻烦了，System Verilog的出现在一定程度上解决了这些问题。

前段时间通过写RISC-V来学习System Verilog，跟着资料动手实践确实能使人进步。

但是随着学习的深入，引导式的资料明显是不够用了，这时候就有必要引入新的学习方法。

对于数字设计来说，只使用testbench明显是不够的，引入几种不同的仿真手段十分有必要。

而处理器是更特别的一种：以状态机为目标的设计使得处理器能从不同的层级进行仿真。于是寻找能提高开发效率的工具和流程就改提上日程了。

刚开始使用各种工具的体验其实都算不上好^^；偶然想起了Verilator，用C++来仿真的方式确实很特别。同时结合自己写过一些嵌入式开发的C语言代码，抱着「从C过渡到C++应该不算太难吧」的想法，W先生就开始了自己的受难之旅（笑）

那么...结果又如何呢？

结果当然是显而易见的，C和C++完全是两回事，更别说是数字电路和软件的联合开发。

套用某些名言，那就是「好复杂的C++，这下便样衰了」。

事已至此，那还是努力学习吧（笑）

# 关于学习C++本身

关于C++的教学视频，国内网络上充斥着太多「零基础学习」之类的内容。成体系的教学视频对于初学者来说固然是好事，但是对于有一定开发经验的人来说，还是过于繁琐了。

我这次学习C++参考的是Cherno的C++教学视频，不涉及过新的标准，但是信息量也足够大，对于有经验却想切换到C++的人来说十分合适。

Link: [Youtube](https://www.youtube.com/watch?v=18c3MTX0PK0&list=PLlrATfBNZ98dudnM48yfGUldqGD0S4FFb), [Bilibili](https://www.bilibili.com/video/BV1Dk4y1j7oj?p=2)

# 关于开发工具的一些感想

一直以来不喜欢商业化工具，尤其是大型的商业化工具，

一直很推崇自由和开源的精神，使用Linux、GCC等工具有益于身心健康，而不是将精力浪费在所谓`在Windows上设置环境变量`和`优化Windows系统`上。
> 当然，在Linux上也一样。I don't use Arch, btw.

另一方面，学校的老师们喜欢让学生们使用付费商业软件，即使他们并不提供授权信息。

在有开源或者免费代替品的情况下，我实在不认为大规模使用破解软件是一件好的事情，无论这些东西是否涉及商用。

但是话又说回来，IC设计毕竟是重生产资料的行业，商业化是主流也是理所当然的，开源生态发展不足也在预料之中。

希望开源生态能乘着RISC-V的这股东风能快速发展吧。
